Endpoint reference

All available endpoints with examples
Overview

Bookings

GET /v1/bookings

List bookings. Query: from, to, status.

Response example:

{"data": [{"id": 1, "guest_name": "John", "check_in": "2025-02-20", "check_out": "2025-02-22", "status": "confirmed"}]}
POST /v1/bookings

Create a booking.

Request body:

{"guest_name": "Jane", "room_id": 5, "check_in": "2025-02-20", "check_out": "2025-02-22"}
GET /v1/bookings/{id}

Get a single booking.

Rooms

GET /v1/rooms

List rooms. Query: status, type_id.

Response example:

{"data": [{"id": 1, "room_number": "101", "type": "Standard", "status": "available"}]}

Restaurant

GET /v1/restaurant/menu

List menu items and categories.

POST /v1/restaurant/orders

Create an order (table_id, items[]).
